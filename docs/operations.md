# Operations

## Updating the System

### Update All Inputs

```shell
cd /etc/nixos
nix flake update
nixos-rebuild switch --flake /etc/nixos#darkhero
```

`nix flake update` rewrites `flake.lock` to the latest commits of all inputs. Review
the diff (`git diff flake.lock`) before rebuilding to understand what changed.

### Update a Single Input

```shell
nix flake lock --update-input nixpkgs
nixos-rebuild switch --flake /etc/nixos#darkhero
```

Prefer partial updates when you need a specific fix or package bump but want to keep
the rest of the system stable. Update all inputs periodically (e.g. monthly) to stay
current with security patches.

## Rollback

NixOS keeps every previous build as a generation in the Nix store. Systemd-boot shows
all generations in the boot menu - selecting an older entry is the easiest recovery path.

### List Generations

```shell
nix-env --list-generations -p /nix/var/nix/profiles/system
```

### Roll Back to the Previous Generation

```shell
nixos-rebuild switch --rollback
```

### Roll Back to a Specific Generation

Boot into the desired generation from the systemd-boot menu (appears on startup).

From a running system, switch the profile pointer and activate:

```shell
nix-env -p /nix/var/nix/profiles/system --switch-generation <N>
/nix/var/nix/profiles/system/bin/switch-to-configuration switch
```

Replace `<N>` with the generation number from the list above.

## Adding a New Host

### 1. Create the Host Directory

```
hosts/<name>/
+-- default.nix                - hostname, boot loader, SOPS age key path
+-- hardware-configuration.nix - generated by nixos-generate-config
`-- impermanence.nix           - persistent directories and files
```

Use `hosts/darkhero/` as a reference. Run `nixos-generate-config` on the target
machine to produce `hardware-configuration.nix`, then copy it into the repo.

### 2. Wire Into flake.nix

```nix
nixosConfigurations = {
  darkhero = mkSystem { ... };

  newhostname = mkSystem {
    system   = "x86_64-linux";
    hostname = "newhostname";
    profiles = [ ./profiles/workstation.nix ];
  };
};
```

### 3. Derive the Host Age Key

On the new machine, extract the age public key from the SSH host key:

```shell
nix-shell -p ssh-to-age --run \
  'ssh-to-age < /etc/ssh/ssh_host_ed25519_key.pub'
```

The host age key file must be placed at the path configured in
`hosts/<name>/default.nix` (`sops.age.keyFile`). Match the path used for
`darkhero` (`/persist/var/lib/sops-nix/key.txt`) or choose a new persistent path
and update the config accordingly.

### 4. Add the Host Key to .sops.yaml and Re-encrypt

```yaml
# .sops.yaml
keys:
  - &host_newhostname age1...   # paste key from step 3
  ...

creation_rules:
  - path_regex: secrets/.*\.sops\.yaml$
    key_groups:
      - age:
          - *host_darkhero_persist
          - *host_newhostname    # add here
          - *user_darkhero
          - *yubikey_vovin
```

Re-encrypt all secret files so the new host key is included:

```shell
sops updatekeys secrets/secrets.sops.yaml
sops updatekeys secrets/ssh.sops.yaml
```

Commit the updated `.sops.yaml` and `secrets/*.sops.yaml` files.

## Test VM

Build a VM image from the current flake config:

```shell
nixos-rebuild build-vm --flake /etc/nixos#darkhero
./result/bin/run-darkhero-vm
```

The VM runs under QEMU with a writable overlay - changes to the VM do not affect the
host. Useful for verifying module syntax, service startup, and basic behaviour.

**Limitations:**

- No access to real hardware (GPU, YubiKey, Bluetooth, specific USB devices).
- SOPS secrets are not decrypted (no age key available in the VM).
- The root filesystem is not ephemeral in the VM - impermanence behaviour differs.
- Display uses the Sway Wayland compositor; requires a Wayland-capable display on the
  host (or set `DISPLAY`/`WAYLAND_DISPLAY` appropriately).

## Garbage Collection

### Manual

Delete all unreferenced store paths and remove old generations:

```shell
nix-collect-garbage -d
```

Run this periodically when disk space is low. The `-d` flag removes old generations
before collecting.

### Automatic

Configured in `modules/system/common.nix` via `nix.gc`:

- Runs **weekly** (systemd timer: `nix-gc.timer`).
- Deletes store paths **older than 30 days** (`--delete-older-than 30d`).

### Store Optimisation

`nix.settings.auto-optimise-store = true` is set in `modules/system/common.nix`.
The store is hard-link deduplicated automatically after every build - no manual action
required.
